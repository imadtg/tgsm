# Docs Index

This is the canonical map of the documentation tree.

Use it to understand which docs describe implemented behavior, which docs describe unresolved choices, and which docs are preserved only as history.

## Start Here

- [Repository README](../README.md): user-facing overview, install, quick start, and the live CLI surface
- [Roadmap](../ROADMAP.md): scope boundaries and the sharp basis of the product
- [Backlog Index](../BACKLOG.md): tracked work and deferred findings outside the roadmap

## Authority Map

- [spec/](./spec/): product and interface contracts
- [operations/](./operations/README.md): current repo and operational behavior
- [decisions/](./decisions/open-decisions.md): unresolved high-leverage choices
- [backlogs/](./backlogs/README.md): tracked work that is not implemented yet
- [proposals/](./proposals/): future product or mutation ideas
- [research/](./research/): evidence, precedents, and external capability notes
- [archive/](./archive/): retired docs preserved for historical context only

Use spec docs for product truth, operations docs for current behavior, backlog and decision docs for tracked but unresolved work, proposal docs for possible future scope, research docs for evidence, and archive docs only when historical context is needed.
