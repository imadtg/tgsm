# Backlogs

Backlog docs track work that matters, but is not part of the current implemented contract.

Use them to separate active operational gaps from broader product usability findings.

## Backlog Split

- [Operations Backlog](./operations.md): install, sync, release, auth, cache, and operator-facing follow-ups
- [Usability Backlog](./usability.md): retrieval and CLI usability shortcomings observed during real usage

## Rule

If an item becomes implemented, move it into the relevant authoritative doc:

- [Operations Overview](../operations/README.md)
- [Security & Auth](../operations/security.md)
- [Product Spec](../spec/product.md)
- [Repository README](../../README.md)
